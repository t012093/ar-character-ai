# AR Character AI - パーソナルアシスタント

## 概要

このプロジェクトは、**OpenAI API (GPT-3/GPT-4) を活用した、AR (拡張現実) で動作するパーソナルアシスタント AI の開発** を目的としています。ユーザーは、AR 空間に表示されるキャラクターと自然な会話を通じて、情報検索、タスク管理、その他のアシスタンス機能を利用できます。

**このリポジトリは、オープンソース版として公開されています。** 有料版では、開発者側で用意した API キーを使用して、ユーザーがより簡単にサービスを利用できるようになります。

**目標:**

*   ユーザーの音声またはテキスト入力を理解し、自然な会話形式で応答する AR キャラクター AI を開発する。
*   OpenAI API を活用して、高度な言語理解と応答生成を実現する。
*   ユーザーのコンテキスト (会話履歴、パーソナル情報など) を考慮した、パーソナライズされたアシスタンスを提供する。
*   オープンソース版と有料版の2つのバージョンを提供し、幅広いユーザーニーズに対応する。
*   将来的には、キャラクターのカスタマイズや、様々な AR デバイスへの対応などを検討する。

## 技術スタック

*   **プログラミング言語:** Python 3.x, JavaScript (TypeScript)
*   **AI フレームワーク/API:**
    *   OpenAI API (GPT-3/GPT-4): 自然言語処理、応答生成
    *   (必要に応じて追加:  Whisper, etc.)
*   **AR 開発:**
    *   Unity (AR Foundation)
    *   (必要に応じて追加: ARKit, ARCore, Vuforia, etc.)
*   **バックエンド:** Supabase
    *   PostgreSQL Database: データベース (会話履歴、ユーザープロファイル、APIキー管理)
    *   Auth: ユーザー認証
    *   Storage: ストレージ (音声データ、画像データ、その他アセット)
    *   Edge Functions: サーバーレスコンピューティング (オープンソース版、有料版)
*   **その他ライブラリ:**
    *   Requests: HTTP リクエストの送信 (Python)
    *   Supabase JS SDK: Supabase クライアントライブラリ

## アーキテクチャ

このシステムは、クライアント (AR アプリ)、Supabase Edge Functions、Supabase Database、OpenAI API で構成されるアーキテクチャを採用する予定です。

**オープンソース版と有料版の違い:**

*   **オープンソース版:** ユーザーが自身の OpenAI API キーを入力して利用する。
*   **有料版:** 開発者が用意した API キーを利用するため、ユーザーはキーの入力が不要。

**データの流れ (オープンソース版):**

1.  ユーザーが AR アプリを通じて、音声またはテキストで指示を入力。
2.  クライアントが Supabase Edge Functions にリクエストを送信 (ユーザーの API キーを含む)。
3.  Edge Functionが、APIキーとユーザー情報をSupabaseのデータベースで検証。
4.  API キーが有効な場合、Edge Function が OpenAI API を呼び出し、ユーザーの入力、会話履歴、ユーザー情報（Supabaseデータベースから取得、オプション）に基づいて応答を生成。
5.  Edge Function が生成した応答 (テキスト、音声) をクライアントに返す。
6.  クライアントが AR キャラクターを通じて、ユーザーに応答を提示。
7.  会話履歴は Supabase データベースに保存。

**データの流れ (有料版):**

1.  ユーザーが AR アプリを通じて、音声またはテキストで指示を入力。
2.  クライアントが Supabase Edge Functions にリクエストを送信。
3.  Edge Function が、環境変数などに設定された開発者側の OpenAI API キーを使用して OpenAI API を呼び出し、ユーザーの入力、会話履歴、ユーザー情報（Supabaseデータベースから取得、オプション）に基づいて応答を生成。
4.  Edge Function が生成した応答 (テキスト、音声) をクライアントに返す。
5.  クライアントが AR キャラクターを通じて、ユーザーに応答を提示。
6.  会話履歴は Supabase データベースに保存。

**コンポーネント:**

*   **AR アプリ (クライアント):** ユーザーとのインターフェース。Unity で開発予定。
*   **Supabase Edge Functions:**
    *   OpenAI API との連携、応答生成。
    *   データベースとのやり取り (会話履歴、ユーザー情報の管理)。
    *   APIキーの認証(オープンソース版)
*   **Supabase Database (PostgreSQL):** データベース (会話履歴、ユーザープロファイル、APIキーの保存)。
*   **Supabase Storage:** ストレージ (音声データ、画像データなど)。
*   **OpenAI API:** 自然言語処理、応答生成。

## 開発計画

1.  **プロトタイプ開発 (1-2 ヶ月):**
    *   OpenAI API との連携の検証 (Supabase Edge Functions を使用)。
    *   基本的な会話機能の実装 (テキストベース)。
    *   会話履歴の保存機能の実装 (Supabase Database)。
    *   オープンソース版の基本的な機能実装
2.  **AR 機能の実装 (2-3 ヶ月):**
    *   Unity を用いた AR キャラクターの表示と基本的なアニメーションの実装。
    *   音声認識と音声合成の統合。
3.  **Supabase 活用基盤の構築 (3-4 ヶ月):**
    *   Supabase Database, Edge Functions, Storage, Auth の設定。
    *   認証機能、APIキー管理などの実装。
    *   オープンソース版と有料版それぞれの Edge Functions の実装
4.  **有料版の開発**:
    *   ユーザー管理、認証、課金システムの開発
    *   開発者側で管理する OpenAI APIキーの設定
5.  **機能拡張とテスト (4 ヶ月以降):**
    *   パーソナライズ機能の強化 (ユーザープロファイルの活用)。
    *   タスク管理などのアシスタンス機能の追加。
    *   ユーザーテストによるフィードバックの収集と改善。
